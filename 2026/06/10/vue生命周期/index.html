vue生命周期

Vue 生命周期简述

Vue 生命周期指组件实例从创建、初始化、挂载、更新到销毁的完整运行过程，过程中会自动执行一系列「生命周期钩子函数」，让开发者可以在特定阶段插入自定义逻辑。
核心阶段与钩子（按执行顺序）

创建阶段（实例初始化，无真实 DOM）

beforeCreate：实例刚创建，响应式数据、方法均未初始化，无法访问 data、methods。
created：实例创建完成，数据、方法已初始化，可正常访问 data/methods；但 DOM 未生成，$el 不可用。常用于发起接口请求、初始化非 DOM 相关数据。
挂载阶段（DOM 首次渲染到页面）
写接口

beforeMount：模板编译完成，真实 DOM 还未挂载到页面前执行。

DOM已经创建，还没渲染，提前请求后端数据，等 DOM 挂载完成后直接渲染，减少页面白屏 / 等待时间。

mounted：组件挂载完成，真实 DOM 已渲染到页面，可直接操作 DOM 节点、初始化第三方 DOM 插件。

更新阶段（响应式数据变化，DOM 重渲染）

beforeUpdate：响应式数据已更新，DOM 重新渲染前触发，可读取更新前的 DOM 状态。
updated：DOM 重渲染完成，可获取数据更新后的最新 DOM 结构。

销毁阶段（实例卸载与资源清理）

beforeUnmount（Vue2 为 beforeDestroy）：实例销毁前执行，实例仍完整可用。常用于清除定时器、解绑全局事件、取消订阅。

unmounted（Vue2 为 destroyed）：实例完全销毁，所有指令解绑、事件监听器移除，子实例同步销毁。
特殊钩子

特殊钩子

activated / deactivated：仅对 <keep-alive> 缓存的组件生效，分别在组件激活、失活时触发。

errorCaptured：捕获子孙组件抛出的错误时触发。

Vue3 组合式 API 专属调试钩子：onRenderTracked（依赖被追踪时触发）、onRenderTriggered（依赖触发更新时触发）。
Vue3 补充说明
选项式 API 钩子与 Vue2 大体一致，仅销毁阶段钩子重命名
。
组合式 API（<script setup>）中以 onXxx 形式调用钩子（如 onMounted）；setup 本身的执行时机早于
beforeCreate，没有对应的 beforeCreate/created 钩子。
