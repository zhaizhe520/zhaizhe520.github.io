AI企劃書

AI企劃書-B站為例

1. 項目概述

1.1 項目名稱

視頻(B站、抖音、Youtube等)字幕提取分析（視頻字幕提取 + 語言分析學習平台）

1.2 項目定位

本項目是一款基於B站視頻資源，整合AI語音識別、自然語言處理（NLP）技術，專注於外語學習的工具類產品。核心解決用戶“利用B站外語視頻（動漫、紀錄片、講座等）高效學習外語”的需求，通過自動提取字幕、深度語言分析、互動學習模式，降低外語精聽、詞彙積累、句型掌握的難度，實現“看視頻、學外語”的場景化學習。

1.3 核心目標

  • 實現視頻鏈接/ 視頻地址解析，自動提取視頻音頻及原有字幕（若有）；

  • 通過AI語音識別技術，將視頻音頻轉寫為精準帶時間軸的字幕（支持多外語）；

  • 對字幕進行深度語言分析，提取詞彙、句型、標註難度，輔助用戶學習；

  • 提供精聽、跟讀、測驗等互動學習模式，強化學習效果；

  • 支持字幕、生詞本、學習報告等內容的導出，便利用戶離線復習。

1.4 目標用戶

外語學習者（以日語、英語、韓語學習者為核心），尤其是喜歡通過B站視頻（動漫、影視、外語講座）進行場景化學習的用戶，涵蓋學生、上班族等不同年齡段，需求聚焦於“精聽練習、詞彙積累、句型掌握”。

1.5 技術核心

B站視頻解析技術、AI語音轉寫（Speech-to-Text）、自然語言處理（NLP）、前後端交互、數據持久化。

2. 總體架構設計

本項目採用“前後端分離”架構，整體分為5大層級，自上而下分別為：用戶交互層、前端應用層、後端服務層、核心技術層、數據存儲層。各層級職責清晰、鬆耦合，便於開發、測試、維護及後續擴展。

2.1 架構總圖

用戶交互層 → 前端應用層 → 後端服務層 → 核心技術層 → 數據存儲層

（附：各層級依賴關係：用戶通過交互層操作前端，前端請求後端服務，後端調用核心技術模塊處理業務，最終將數據存儲至數據層，並將結果反饋至前端展示給用戶。）

2.2 各層級核心職責

層級 核心職責 核心組件/技術
用戶交互層 提供用戶操作入口，展示業務數據及交互界面，接收用戶操作指令 網頁端（PC/移動端適配）、按鈕、輸入框、視頻播放器、字幕展示區
前端應用層 處理前端頁面渲染、用戶交互邏輯、請求封裝、數據展示，與後端進行通信 HTML、CSS、JavaScript、Vue.js（簡化開發）、Axios（請求工具）
後端服務層 接收前端請求，調度核心技術模塊，處理業務邏輯，返回處理結果 Python、FastAPI（輕量高效）、接口路由、請求校驗、響應封裝
核心技術層 實現項目核心功能，包括視頻解析、語音轉寫、語言分析、學習模式邏輯 B站解析工具、OpenAI Whisper、NLP工具（fugashi、nltk）、音頻處理工具
數據存儲層 存儲用戶數據、歷史記錄、生詞本、字幕數據等，提供數據查詢、新增、刪除、修改能力 SQLite（輕量級，便於部署）、文件存儲（字幕文件、音頻文件）

3. 詳細模塊設計

基於總體架構，將項目拆分為6個核心模塊，每個模塊對應具體業務功能，模塊間通過接口通信，確保業務流程順暢。

3.1 用戶交互模塊（用戶交互層+前端應用層）

3.1.1 模塊功能

負責用戶與系統的所有交互，展示業務數據，接收用戶操作，並將操作指令傳遞至後端，同時展示後端返回的處理結果。

3.1.2 核心子模塊

  • 視頻輸入子模塊：提供B站視頻鏈接/BV號輸入框、提交按鈕，支持鏈接校驗（判斷是否為有效B站鏈接）；

  • 視頻播放子模塊：集成視頻播放器，支持播放、暫停、拖動進度，並同步顯示對應時間軸的字幕；

  • 字幕展示子模塊：展示提取/轉寫的字幕，支持切換外語原字幕、雙語字幕（外語+中文），點擊字幕可定位到對應視頻片段；

  • 語言分析展示子模塊：展示詞彙分析（詞性、頻率、難度）、句型提取、生詞列表，支持點擊詞彙查看詳細解釋；

  • 學習模式子模塊：提供精聽、跟讀、測驗三種模式的入口，展示對應模式的交互界面（如挖空填空、語音錄製）；

  • 導出子模塊：提供字幕（SRT/TXT）、生詞本（Excel/Anki）、學習報告的導出按鈕，支持選擇導出格式；

  • 用戶中心子模塊（簡易版）：記錄用戶歷史操作、收藏生詞、學習記錄，支持清除歷史、管理生詞本。

3.1.3 技術實現

採用HTML+CSS+JavaScript搭建基礎頁面，使用Vue.js簡化頁面渲染和數據綁定，確保頁面響應式（適配PC端和移動端）；通過Axios封裝前端請求，與後端接口對接，處理請求狀態（加載中、成功、失敗）；集成視頻播放插件（如video.js），實現視頻播放與字幕同步。

3.2 B站視頻解析模塊（核心技術層）

3.2.1 模塊功能

接收前端傳遞的B站視頻鏈接/BV號，解析視頻信息，獲取視頻音頻文件（用於語音轉寫）和原有字幕文件（若有），並對音頻、字幕進行預處理，為後續語音轉寫和語言分析提供數據支持。

3.2.2 核心子模塊

  • 鏈接解析子模塊：解析B站視頻鏈接/BV號，獲取視頻唯一標識，調用解析工具獲取視頻播放地址、音頻地址、原有字幕地址（若有）；

  • 音頻下載與預處理子模塊：下載視頻對應的音頻文件（轉換為mp3格式，降低體積），對音頻進行降噪、截取（若用戶需要部分片段）處理；

  • 原有字幕提取子模塊：若B站視頻有對應外語字幕（如日語、英語），提取字幕文件（通常為ass/srt格式），轉換為統一的文本格式，並對字幕進行去雜、時間軸校準；

  • 數據校驗子模塊：校驗下載的音頻、字幕文件的完整性，若文件損壞或解析失敗，返回錯誤信息至後端，由後端反饋至前端。

3.2.3 技術實現

使用Python的you-get或yt-dlp工具，解析B站視頻鏈接，下載音頻文件；使用ffmpeg工具將音頻轉換為mp3格式，並進行降噪處理；解析ass/srt字幕文件，使用python-srt庫將字幕轉換為文本格式，提取字幕內容和時間軸信息；若解析失敗，返回對應的錯誤碼（如“鏈接無效”“視頻無法下載”）。

3.3 AI語音轉寫（字幕生成）模塊（核心技術層）

3.3.1 模塊功能

當B站視頻無原有外語字幕時，接收視頻解析模塊傳遞的音頻文件，通過AI語音識別技術，將音頻轉寫為帶時間軸的字幕文本，支持多種外語（日語、英語、韓語等），確保字幕的精準度和時間軸的匹配度。

3.3.2 核心子模塊

  • 語言識別子模塊：自動識別音頻中的語言類型（如日語、英語），若用戶手動指定語言，優先使用用戶指定的語言；

  • 語音轉寫子模塊：調用AI語音識別接口/工具，將音頻文件轉寫為文本，並生成對應的時間軸（每句字幕的開始時間和結束時間）；

  • 字幕優化子模塊：對轉寫的字幕進行校準（修正語法錯誤、語義偏差），調整時間軸，確保字幕與視頻音頻同步；

  • 字幕格式轉換子模塊：將轉寫后的字幕轉換為SRT、TXT兩種格式，便於展示和導出。

3.3.3 技術實現

優先使用OpenAI Whisper（免費、高精度、支持多語言），通過Python調用Whisper庫，加載base/large模型（根據精準度需求選擇，large模型精準度更高），對mp3音頻文件進行轉寫；設置語言參數（如–language ja/en），確保轉寫準確性；使用python-srt庫生成SRT格式字幕，並對時間軸進行微調；若用戶需要更高穩定性，可替換為百度AI、騰訊雲語音識別API（付費，穩定性更高）。

3.4 語言分析模塊（核心技術層）

3.4.1 模塊功能

接收字幕生成模塊傳遞的字幕文本（原有字幕或AI轉寫字幕），通過NLP技術對字幕進行深度分析，提取詞彙、句型、標註難度，為用戶提供學習參考，核心是輔助用戶掌握字幕中的外語知識。

3.4.2 核心子模塊

  • 分詞子模塊：對字幕文本進行分詞（日語使用fugashi+unidic，英語使用nltk/spacy），拆分出單詞、短語，去除無意義詞（如助詞、標點）；

  • 詞彙分析子模塊：標註每個詞彙的詞性（名詞、動詞、形容詞等）、發音（日語假名、英語音標）、中文釋義，統計每個詞彙在字幕中的出現頻率；

  • 難度標註子模塊：根據詞彙、句型的複雜度，標註難度級別（日語N5N1，英語CEFR A1C2），便於用戶根據自身水平篩選學習內容；

  • 句型提取子模塊：提取字幕中反覆出現的常用句型、固定短語，標註句型結構、用法，並給出例句（來自字幕本身）；

  • 生詞篩選子模塊：根據用戶設定的學習水平，自動篩選出超出用戶水平的生詞，生成生詞列表，便於用戶重點學習。

3.4.3 技術實現

日語分詞使用fugashi庫配合unidic詞典，實現精確分詞和詞性標註；英語分詞使用nltk或spacy庫，支持詞性標註和詞義解析；詞彙難度標註通過對比預定義的詞彙難度詞典（如日語N5~N1詞典、英語CEFR詞典）實現；句型提取使用規則匹配+統計方法，識別字幕中反覆出現的句式結構；所有分析結果整理為標準數據格式，傳遞至後端服務層。

3.5 學習模式模塊（核心技術層+後端服務層）

3.5.1 模塊功能

基於字幕和語言分析結果，提供三種核心學習模式，強化用戶的聽力、口語、詞彙掌握能力，實現“學練結合”。

3.5.2 核心子模塊

  • 精聽模式子模塊：用戶點擊任意字幕句子，自動播放對應的視頻片段（可設置循環播放），支持隱藏字幕、顯示字幕切換，幫助用戶練習聽力；

  • 跟讀模式子模塊：播放字幕對應的音頻片段，用戶可錄製自己的跟讀語音，系統通過語音識別對比用戶語音與原音的相似度，給出評分和改進建議；

  • 測驗模式子模塊：基於字幕中的詞彙、句型，自動生成測驗題（挖空填空、單詞翻譯、句型仿写），用戶完成測驗後，給出分數和解析，強化記憶；

  • 學習記錄子模塊：記錄用戶在各種模式下的學習時間、學習內容、測驗成績，生成簡單的學習報告，便於用戶瞭解自身學習情況。

3.5.3 技術實現

精聽模式通過前端視頻播放器的時間軸控制，綁定字幕點擊事件，實現片段播放和循環；跟讀模式集成語音錄製工具（如WebRTC），錄製用戶語音，調用Whisper API對用戶語音進行轉寫，與原字幕文本對比，計算相似度；測驗模式通過後端代碼生成試題（基於語言分析模塊的詞彙、句型數據），前端展示試題，用戶提交答案後，後端進行評分並返回解析；學習記錄通過數據存儲層記錄，並生成簡單的統計報告。

3.6 數據存儲與導出模塊（數據存儲層+後端服務層）

3.6.1 模塊功能

負責項目所有數據的存儲、管理和導出，包括用戶數據、字幕數據、語言分析數據、學習記錄等，確保數據的安全性和可訪問性，同時支持用戶導出所需內容，便於離線復習。

3.6.2 核心子模塊

  • 數據存儲子模塊：使用SQLite數據庫，存儲用戶信息（簡易版，無注冊登錄，僅存本地歷史）、視頻解析記錄、字幕數據、生詞本、學習記錄；使用文件存儲，保存下載的音頻文件、字幕文件；

  • 數據管理子模塊：負責數據的新增、查詢、刪除、修改，如用戶添加/刪除生詞、清除歷史記錄、更新學習記錄；

  • 導出子模塊：接收前端導出請求，根據用戶選擇的格式（SRT/TXT/Excel/Anki），從數據庫或文件中提取對應數據，生成導出文件，返回給前端用戶；

  • 數據備份子模塊（簡易版）：定期備份SQLite數據庫和重要文件，防止數據丟失。

3.6.3 技術實現

使用Python的sqlite3庫操作SQLite數據庫，設計數據表（用戶表、視頻記錄表、字幕表、生詞本表、學習記錄表），實現數據的CRUD操作；文件存儲使用本地文件目錄，按“視頻BV號”分類存儲音頻、字幕文件；導出功能使用python-docx（Excel）、python-srt（SRT）等庫，生成對應格式的文件，通過後端接口返回給前端，供用戶下載；數據備份通過定時任務（如APScheduler）實現，定期複製數據庫文件和文件目錄。

4. 後端接口設計

後端基於FastAPI搭建，提供RESTful接口，供前端調用，所有接口返回JSON格式數據，包含狀態碼、消息、數據三部分，確保接口的規範性和可維護性。

4.1 接口通用格式

請求方式：GET/POST（根據業務場景選擇，查詢用GET，提交/修改用POST）；

返回格式：{“code”: 狀態碼, “message”: “提示信息”, “data”: 業務數據}

狀態碼說明：200（成功）、400（請求參數錯誤）、404（資源不存在）、500（服務器錯誤）。

4.2 核心接口列表

接口路徑 請求方式 核心功能 請求參數 返回數據
/api/video/parse POST 解析B站視頻鏈接/BV號，獲取音頻、原有字幕 video_url（視頻鏈接/BV號） 音頻地址、原有字幕（若有）、視頻標題
/api/subtitle/generate POST AI語音轉寫，生成字幕 audio_path（音頻路徑）、language（語言） SRT字幕文本、TXT字幕文本、時間軸信息
/api/language/analyze POST 對字幕進行語言分析（詞彙、句型、難度） subtitle_text（字幕文本）、language（語言） 詞彙列表、句型列表、生詞列表、難度分佈
/api/learning/quiz POST 生成測驗題（挖空、翻譯） vocab_list（詞彙列表）、sentence_list（句型列表） 試題列表、正確答案、解析
/api/learning/record POST 記錄用戶學習記錄 video_id（視頻標識）、learning_type（學習模式）、score（測驗分數） 記錄成功標識、學習時間
/api/export/subtitle GET 導出字幕文件（SRT/TXT） subtitle_id（字幕標識）、format（導出格式） 文件下載鏈接
/api/export/vocab GET 導出生詞本（Excel/Anki） vocab_id（生詞本標識）、format（導出格式） 文件下載鏈接
/api/history/list GET 查詢用戶學習歷史 page（頁數）、size（每頁條數） 歷史記錄列表、總條數

5. 數據庫設計

本項目使用SQLite數據庫（輕量級、無需單獨部署、適合小型項目），設計5張核心數據表，分別對應用戶、視頻記錄、字幕、生詞本、學習記錄，表結構如下（字段說明：字段名、類型、是否主鍵、備註）。

5.1 用戶表（user）

簡易版用戶表，無注冊登錄，僅存本地用戶信息（若後續擴展可增加注冊登錄功能）。

字段名 類型 主鍵 備註
user_id INTEGER 是 用戶唯一標識，自增
user_name TEXT 否 用戶名（默認“默認用戶”）
learning_level TEXT 否 用戶學習水平（如日語N3、英語B1）
create_time TIMESTAMP 否 用戶創建時間（默認當前時間）

5.2 視頻記錄表（video_record）

記錄用戶解析過的B站視頻信息。

字段名 類型 主鍵 備註
video_id INTEGER 是 視頻唯一標識，自增
bv_id TEXT 否 B站視頻BV號（唯一）
video_title TEXT 否 視頻標題
video_url TEXT 否 B站視頻鏈接
audio_path TEXT 否 音頻文件本地存儲路徑
user_id INTEGER 否 關聯用戶表user_id
parse_time TIMESTAMP 否 解析時間

5.3 字幕表（subtitle）

記錄生成的字幕信息。

字段名 類型 主鍵 備註
subtitle_id INTEGER 是 字幕唯一標識，自增
video_id INTEGER 否 關聯視頻記錄表video_id
language TEXT 否 字幕語言（ja/en/ko等）
